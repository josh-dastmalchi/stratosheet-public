Table Columns

Columns define what kind of data is stored, and what kind of validation should be performed on the data.

Each column in a table must have a unique name.

General Columns

The most important part of a column is its column type. The column type defines what kind of information can be stored in the column, how it is displayed, and what kind of validation is available.

Single-Line Text

Single-line text columns can store any kind of text data, such as names, email addresses, or phone numbers.

This type of column can have a minimum or maximum length, and also supports special validation for the following kinds of data:

  • Credit Card Numbers
  • Email Addresses
  • ZIP Codes
  • URLs

Multi-Line Text

Multi-line text columns can store any kind of text data, and can span multiple lines. Examples include descriptions, notes, or comments.

This type of column can have a minimum length, maximum length, or both.

Number

Number columns can store any kind of numerical data, such as measurements, currency, or quantities.

This type of column can have a minimum value, maximum value, or both.

This type of column supports formatting to show a currency symbol or to specify the number of decimal places.

True / False

True / false columns store a logical true/false value. Examples include being active, being on, etc.

Attachment

Attachment columns can store files such as images or documents.

Dates and Times

Date and Time

Date and Time columns store both a date and a time. This type of data is used to represent a specific point in time, such as an event's start time.

These columns must specify a time zone.

These columns can be formatted to various standard formats, and can use either a 12- or 24-hour clock.

Date Only

Date Only columns are used to store a date without a time. This type of data is used to represent a specific day, such as a birthday or a due date.

These columns must specify a time zone.

These columns can be formatted to various standard formats.

Time Only

Time Only columns are used to store a time without a date. This type of data is used to represent a specific time of day, such as a recurring meeting time or a store's opening hours.

These columns can use either a 12- or 24-hour clock.

References

Row Reference

Row Reference columns are used to reference another row in the same or a different table. This type of data is used to create relationships between tables, such as a customer's orders or a project's tasks.

User Reference

User Reference columns are used to reference a user in the workspace. This type of data is used to assign tasks, track changes, or send notifications to specific users.
